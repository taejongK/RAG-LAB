## 1. 데이터 개념

데이터 분석에 사용되는 자원으로써, 분석가의 로컬 PC 내에 존재하는 텍스트 및 이미지 파일이나 데이

터베이스에 존재하는 다양한 형태의 리소스를 의미합니다

분석 알고리즘 수행에 필요한 이러한 데이터들은 수행 전에 등록되어야만 사용할 수 있으며,

파일의 경우, 업로드 기능을 이용해서 등록할 수 있고, 데이터베이스에 존재하는 데이터의 경우, DB 커넥

션 정보를 이용하여 등록할 수 있습니다

## 2. 데이터 카테고리

데이터는 파일 위치, DB 접속정보, 사용자 함수 세가지 유형의 카테고리로 나뉩니다


-----

![](/home/taejong_kim/workspace/rag-lab/database/image/BA_사용메뉴얼_분석가.pdf-12-1.png)

**파일** **위치: 분석에** 사용되는 파일 유형의 데이터를 등록하는 영역으로, 분석가 로컬 PC의 파일을 등

록하거나 분석 실행의 결과물이 위치하게 되며, 필요에 따라 서브 카테고리를 생성하여 관리할 수

있습니다

**DB 접속정보: 데이터베이스에** 존재하는 데이터를 수집하기 위한 DB 접속정보를 등록하는 영역으

로, 해당 접속 정보를 이용하여 데이터베이스로부터 데이터를 추출하는 노드를 작성하게 되며, 데

이터 노드는 관리자만 작성할 수 있습니다


![](/home/taejong_kim/workspace/rag-lab/database/image/BA_사용메뉴얼_분석가.pdf-12-0.png)

-----

![](/home/taejong_kim/workspace/rag-lab/database/image/BA_사용메뉴얼_분석가.pdf-13-1.png)

**사용자** **함수: 알고리즘** 내에서 사용되는 R 또는 Python으로 작성된 사용자정의 패키지를 등록하는

영역으로, 이 곳에 등록된 패키지들은 알고리즘 내에서 import를 통해 사용되어지며, 관리자만 등

록할 수 있습니다

## 1. 분석플로우 개념

데이터 분석을 위한 일련의 과정을 표현하는 흐름의 단위로써, 알고리즘들의 조합으로 구성됩니다

분석플로우를 구성하는 알고리즘의 최소 개수는 1개이며, 두 개 이상의 알고리즘을 연결하여 하나의 분

석플로우를 작성할 수 있는데 이 때, 첫 번째 알고리즘 Output의 Data Type과 두 번째 알고리즘 Input의

Data Type은 일치해야 합니다

류 체계는 관리자 이외의 사용자가 변경하는 것을 허용하지 않습니다

## 1. IF문 기본개념

특정 알고리즘의 Output에 따라 분석플로우의 진행을 분기할 수 있는 분석플로우의 기능입니다

IF문 앞 알고리즘의 Output을 Input으로 받아 평가합니다

IF에서는 새로운 Output을 생성할 수 없고 앞에서 받은 Input을 조건의 결과(True / False)에 따라

다음 알고리즘의 Input으로 넘길 수 있습니다

## 2. IF 사용방법

1. IF 신규 **생성**

IF를 드래그하여 삽입할 곳에 끌어넣습니다.

편집창 내의 IF를 클릭하면 하단에 조건 편집창이 생성됩니다. Edit 버튼을 클릭하여 조건 편집

창을 활성화 할 수 있습니다.


![](/home/taejong_kim/workspace/rag-lab/database/image/BA_사용메뉴얼_분석가.pdf-158-0.png)

-----

![](/home/taejong_kim/workspace/rag-lab/database/image/BA_사용메뉴얼_분석가.pdf-159-0.png)

Input과 Condition Variable을 선언하고 스크립트를 작성합니다.예제에서는 DataFrame의

row 개수에 대한 조건절을 작성했습니다.

Input : 앞에서 연결되는 알고리즘의 Output중 조건절로 판단할 Output과 같은 변수 타

입으로 선언합니다.

Condition Variable : IF 실행의 결과로 True 또는 False 값이 담길 변수명입니다.

Save 버튼을 클릭하면 IF에 연결점이 생기는 것을 확인할 수 있습니다.

IF는 알고리즘이 아닌 분석플로우 기능이기 때문에 반드시 분석플로우 저장을 해 주셔야 IF의

수정사항이 저장이 됩니다.

## 1. SWITCH문 기본개념

SWITCH문 앞 알고리즘의 Output을 Input으로 받아 평가합니다.

SWITCH에서는 새로운 Output을 생성할 수 없고 앞에서 받은 Input을 조건의 결과(조건 변수의 값)

에 따라 다음 알고리즘의 Input으로 넘길 수 있습니다.

## 2. SWITCH 사용방법

건 편집창을 활성화 할 수 있습니다.

Condition Variable : SWITCH 실행의 결과로 Switch Condition에 선언된 값 중 하나가

평소에 사용하고 싶은 패키지가 있었는데, BA에서 제공하지 않는 패키지 혹은 버전이라서 곤란하신 경

험이 있으셨나요?

이제는 하나의 워크플로우에서, 이전보다 다양한 패키지를 참조할 수 있도록 라이브러리별 실행 엔진을

설정하는 기능이 추가되었습니다

## 1. 실행 엔진 개념

관리자에 의해 생성되어 특정 패키지들이 설치된 커스텀한 실행 엔진을 의미합니다

스크립트에 작성한 패키지 버전을 기준으로 매핑해준다면, 워크플로우 실행시 해당 정보를 참조해 실행

환경이 바뀝니다

## 2. 실행 엔진 정보 조회

왼쪽의 워크플로우 검색 화면에서 실행 엔진 정보를 조회할 워크플로우를 우클릭 후, Open 혹은 Detail

을 클릭합니다

Open을 선택한 경우 에디터 상단의 컨트롤바에서 Workflow Detail을 클릭합니다

Workflow Detail창의 최하단에서 Engine 정보를 조회할 수 있습니다

Engine정보에서 해당 워크플로우에 포함된 테스크의 정보를 간략히 확인할 수 있습니다

테스크 정보에서 if/switch 및 java라이브러리의 정보는 포함되지 않습니다

워크플로우 디테일하단에 위치한 Edit버튼을 클릭합니다

실행 엔진 정보 설정란에서 다음과 같은 기능을 이용할 수 있습니다

1. 동일한 언어군의 테스크별 엔진을 일괄 설정 기능

2. 테스크 단위의 실행 엔진 정보 단일 설정 기능

해당 기능을 이용해 실행 엔진 정보를 업데이트해봅시다

먼저, 동일한 언어군의 엔진 정보 일괄 설정 기능을 사용해 보겠습니다

## 1. 배치 분석플로우 개념

작성된 분석플로우를 특정 주기 (일, 시, 분)에 지속적으로 실행시켜 원하는 결과를 얻기 위해서는 배치

기능을 활용할 수 있습니다

배치 잡으로 등록된 분석플로우 내에서 분석에 필요한 데이터를 가변적으로 수집하고자 할 때나 분석플

로우의 실행 정보를 이용하여 알고리즘의 실행에 활용하고자 할 때는 별도의 추가 작업을 필요로 합니다

즉, 별도의 인자 값을 통해서 분석플로우의 실행을 온라인과 배치로 구분하고, 실행 시 분석플로우의 흐

름 및 스크립트의 동적인 실행을 가능하게 할 수 있습니다

## 2. 배치로 분석플로우 실행 시 제약 사항

**콘솔** **메시지** **확인** **불가능**

분석플로우 편집기를 통한 실행의 경우, 하단 정보영역에 print문과 에러 메시지 등으로 출력되는 콘솔

메시지를 확인 할 수 있지만

배치로 실행되는 경우, 메시지의 확인이 불가능하고 단지 알고리즘에 선언된 Output 변수의 값만 확인

이 가능합니다

분석플로우 편집기를 통한 결과 확인

오류가 발생한 알고리즘은 아이콘으로 표시되며, 하단 정보영역에서 콘솔 메시지를 확인 할 수 있

습니다

## 1. 설명

이전 알고리즘 노드의 Output 변수에 값이 없는 경우, 다음 알고리즘 노드의 Input 변수에 값이 전달되

지 않게되는데, Python 및 R에서 제공되는 globals() 함수와 ls() 함수를 이용해서 데이터의 존재 유무를

```
# Python 스크립트
if 'Input' not in globals():
  # 예외처리
# R 스크립트
if(!('Input' %in% ls())) {
  # 예외처리
}

```
